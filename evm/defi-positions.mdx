![Defi Positions API](/images/evm/defi-positions.svg)

The DeFi Positions API returns a wallet's active positions across liquidity pools, lending protocols, yield strategies, and tokenized DeFi assets. The endpoint is available at the beta path (`/beta/evm/defi/positions`). Each position includes token holdings, USD valuations, underlying asset metadata, and protocol-specific details such as tick ranges, collateral status, or reward accruals.

## Response Structure

Every response contains a `positions` array and an `aggregations` object:

```json
{
  "positions": [...],
  "aggregations": {
    "total_value_usd": 4201.34,
    "total_by_chain": {
      "1": 3800.00,
      "8453": 401.34
    }
  }
}
```

Each entry in `positions` has a `type` discriminator field that determines its shape. All positions also include `chain` (string name, e.g. `"ethereum"`) and `chain_id` (numeric). Token metadata is returned as **nested objects** — not flat fields.

### Liquidity Pool Positions

Liquidity pool positions represent token pairs deposited into automated market makers (AMMs).

<AccordionGroup>

### Lending Positions

Lending positions include supplied collateral and outstanding debt across major lending protocols.

<AccordionGroup>

  **Type:** `Tokenized`  
  **Token Types:** `AtokenV2`, `AaveV2VariableDebt`  
  **Chains:** Ethereum (1), Polygon (137), Avalanche (43114)  
  **Description:** Supply positions (aTokens) and variable debt positions on Aave v2. The `token` object is the protocol token (e.g. `aWETH`). ### Governance Positions

Governance positions represent tokens locked in vote-escrow contracts for protocol governance participation.

<AccordionGroup>
  <Accordion title="VeNFT (Vote-Escrow)" icon="lock">
  **Type:** `VeNft`
  **Chains:** Ethereum (1), Base (8453), Optimism (10)
  **Description:** Vote-escrow NFT positions for governance tokens (e.g. veAERO on Base, veVELO on Optimism, veCRV on Ethereum). The `ve_contract` is the vote-escrow contract; `token_id` is the NFT identifier (hex). `underlying_token` is a full token info object. `locked_amount` is the escrowed token balance; `lock_end` is the unlock timestamp (Unix seconds); `is_permanent` indicates a perpetual lock. ## Compute Units

Each request consumes **10 Compute Units per processed chain ID**. Filtering to fewer chains lowers usage. Aggregations are calculated server-side and included in the same response.

<Note>
To learn more specifics, please visit the [Compute Units](/compute-units) page.
</Note>
