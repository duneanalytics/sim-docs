The Single Balance sub-endpoint returns a wallet’s balance for one token on one chain. Use this when you need a precise balance for a specific asset rather than a wallet's full portfolio.

<Note>
  This endpoint accepts exactly **one chain** via the `chain_ids` query parameter. It does not accept tags or multiple chain IDs.
</Note>

## ERC-20 tokens vs native assets

To retrieve an ERC‑20 token, replace `{token_address}` with the ERC‑20 contract address.
To retrieve a native token balance, set `{token_address}` to `native` in the same path.

## Compute Unit Cost

This endpoint has a fixed CU cost of **1** per request. It accepts exactly one `chain_ids` value. See the [Compute Units](/compute-units) page for details.
