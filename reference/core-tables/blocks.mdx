# Usage notes

The table contains only one row per block number, reorg'd blocks are removed.

Some columns are optional since they contain fields that were added at a later time in the chain. To properly handle null values, pinot suggests using`SET enableNullHandling=true;` before the SQL query. That will enable filtering the null values using the`WHERE col IS NULL` predicate and ignore null values in aggregation functions.

[Here](https://docs.pinot.apache.org/developers/advanced/null-value-support#advanced-null-handling-support), you can learn more about how Pinot handles null values in queries.
